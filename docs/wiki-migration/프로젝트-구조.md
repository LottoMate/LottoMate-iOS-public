# 프로젝트 구조

## Overview
LottoMate는 UIKit + ReactorKit 기반 iOS 앱이며, 기능 단위로 `Sources` 아래를 분리해 관리합니다.

## Directory Structure
```text
LottoMate/
├── LottoMate/
│   ├── Application/              # 앱 진입점 및 초기 흐름
│   ├── Resources/                # 에셋, plist, 스토리보드
│   └── Sources/
│       ├── Common/               # Extensions, Managers, Utils
│       ├── Data/
│       │   └── Network/
│       │       ├── DTOs          # API 응답 모델
│       │       ├── Endpoints     # Moya TargetType
│       │       └── Services      # 네트워크 서비스
│       ├── Domain/
│       │   └── Entities          # 도메인 모델
│       ├── Features/             # 화면/기능 단위 UI + 로직
│       │   ├── Home
│       │   ├── Storage
│       │   ├── Map
│       │   ├── WinningReview
│       │   ├── WinnerGuide
│       │   ├── WinningInfoDetail
│       │   ├── Login
│       │   ├── Lounge
│       │   ├── QR
│       │   ├── Onboarding
│       │   └── Setting
│       ├── Models/               # ViewModel/Reactor 보조 모델
│       └── Views/                # 공통 UI 뷰/컴포넌트
├── LottoMateTests/               # XCTest
├── LottoMateUITests/             # UI 테스트
├── Podfile
└── Pods/
```

## 레이어 역할
- `Application`: AppDelegate/SceneDelegate, 앱 시작 플로우
- `Features`: 화면 중심 비즈니스 로직(Reactor 포함)
- `Data/Network`: API 통신, endpoint, DTO, service
- `Common`: 앱 전역 유틸/매니저
- `Views`: 재사용 UI 컴포넌트

## 현재 공개 버전 기준
- 기본 데이터 모드: `mock`
- Firebase Remote Config 미사용
- 업데이트 버전 정책: `Info.plist`의 로컬 키 사용

## 시작하기
1. Xcode에서 `LottoMate.xcworkspace` 열기
2. `Config.xcconfig.example` 복사 후 `Config.xcconfig` 생성
3. `NMF_CLIENT_ID` 설정
4. 빌드/실행
